📋Purpose & Scope

The STC AI Agent Dashboard is the single source of truth for all AI agents built by the team. It shows who built what, what it automates, the skill rating, the gap plan, and what's next for each person.

This SOP covers how every team member — PH and US — accesses the dashboard, submits their agents, and keeps their cards updated as they continue building.

Scope: This SOP applies to all seven STC team members — Rizza, Charlene, Ken, PJ, Holley, David, and Gonzalo. Both training agents (Days 1-17) and all post-training agents belong on this board.

Every team member submits one Google Form entry per agent. If you built three agents, submit the form three times.

Important: Submit one form per agent — not one form for all your agents combined. The dashboard creates one card per submission.

🔄Refreshing Data

The dashboard does not auto-refresh. Data must be refreshed manually after a new form submission.

1
Click the Refresh button after any form submission
Top right of the dashboard. Always click this after a team member notifies you they submitted — not before.
Rizza Infante
2
Wait 2 to 5 seconds for cards to reload
The dashboard fetches the published CSV from Google Sheets. It is fast but not instant.
3
If data does not load, hard refresh the browser
Press Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows) to force a full reload. Then click Refresh again.

⭐Getting Your AI Rating

The AI Rating is not automatic. Each team member must prompt Claude themselves and use the number Claude returns.

1
Open a new Claude conversation
Do not use an existing conversation — start fresh so Claude has no prior context that could influence the rating.
2
Paste the rating prompt with your agent details
"I built an AI agent called [Agent Name]. It [describe what it does in 1-2 sentences]. I deployed it [how — Claude Project only / Vercel / GitHub / etc.]. It [current status — is in daily use / is still being built / handles X tasks per week]. Based on what I built and how I built it, rate my agentic AI skill level from 1 to 10. Be honest and specific about what I did well and what would need to improve for a higher score."
3
Use the number Claude gives you as your AI Rating in the form
If Claude gives a range (e.g., "6 to 7"), use the lower number. Honest ratings are more useful than optimistic ones.
Rizza's role: After all team members submit, Rizza reviews each AI Rating for consistency. If a rating seems significantly off, she re-runs the prompt and updates the Google Sheet directly.

✨EBI — Even Better If

This is not optional. EBI is one of the most important skills in the program. Every team member must add the EBI skill to their Claude before submitting their agent card. It directly impacts the quality and rating of everything you build.

EBI stands for Even Better If — a structured feedback process used to continuously improve your AI agent. Instead of just accepting the first output Claude gives you, the EBI process pushes you to identify what's working, what could be stronger, and apply targeted improvements in order of impact.

Why EBI Matters for Your Agent

The gap between a 5/10 agent and a 7/10 agent is almost always EBI. Agents that were run through the EBI process consistently score higher on output quality, prompt architecture, and feature completeness. It is the single most effective tool for closing the gap between your self-rating and your AI rating.

Without EBIWith EBI
First output accepted as finalOutput reviewed, ranked, and improved before finalizing
Vague prompt improvements over timeSpecific, ranked improvements applied in order of impact
Agent plateaus at initial qualityAgent gets measurably better with each iteration
EBI field on dashboard shows No or AwaitingEBI field shows Yes — signals a higher-quality build

How to Add EBI to Your Claude

1
Open your Claude Project for your agent
Go to claude.ai, open your agent's Project. The EBI skill is added as part of your system prompt or as a saved instruction.
All team members
2
Add the EBI skill to your Claude — paste this into a new conversation in your Project
EBI Process — run this on any output I give you before we finalize it: 1. What's working — name what's strong and should be preserved 2. Even Better If... — list 5 to 10 ranked improvements, sharpest impact first, each starting with "Even better if..." 3. Apply — when I pick numbers, rewrite with only those changes applied 4. Go deeper — after applying, offer one more EBI pass at the next level Format: clean, numbered, no fluff. Think like an operator reviewing work before it ships.
All team members
3
Run EBI on your agent's most important output
After your agent produces its main output — the triage decision, the payroll report, the meeting summary — paste it back and say "Run EBI on this." Pick the improvements that matter most and apply them.
All team members
4
Run EBI on your system prompt
Paste your full agent system prompt and say "Run EBI on this system prompt — what would make this agent more precise, reliable, and useful?" Apply the top 3 suggestions.
All team members
5
Mark EBI Skill Applied as Yes in the form
When you submit your agent card, select Yes for EBI Skill Applied and write a short note in EBI Notes describing what you improved. Example: "Used EBI to tighten the routing logic and improve output format clarity."
All team members

When to Run EBI

TriggerWhat to EBI
After your agent produces its first outputThe output itself — is it clear, complete, and usable?
After writing your system promptThe system prompt — is it precise and unambiguous?
Before marking your agent as LiveThe full agent — would a teammate understand how to use it?
When your AI Rating is lower than your self-ratingWhatever Claude flags as the weakest area in its rating feedback
Every time you add a new feature or workflowThe new addition — does it integrate cleanly with what already exists?
If your EBI field shows "No" or "Awaiting" on the dashboard: Your card is flagged. Rizza will follow up. Run EBI on your agent, resubmit the form with Yes selected, and include what you improved in the EBI Notes field.

🚀Adding Post-Training Agents

Any agent built after the 17-day training program belongs on this board. The process is identical to submitting a training agent — the only difference is the Phase field.

Phase field: Select Post-Training for any agent built after June 9, 2026. The card will show an amber rocket tag instead of the green graduation cap.

There is no deadline for post-training submissions. Submit whenever a new agent is built or when an existing agent reaches a significant new milestone (e.g., went from In Build to Live).

📝Notes

Google Sheet row 1 rule: Never add helper text back to row 1 of the Sheet. The dashboard reads column headers by exact name. If headers contain extra text, cards will show "Unknown" and fields will not map correctly.
CSV publishing: The dashboard reads from the Google Sheet via a published CSV URL. If the Sheet is ever unpublished, the dashboard will stop loading. To re-publish: File → Share → Publish to web → Sheet1 → CSV → Publish.
File naming: This SOP is saved in Google Drive as: 2026-06-23 STC AI Agent Dashboard SOP v1.0
